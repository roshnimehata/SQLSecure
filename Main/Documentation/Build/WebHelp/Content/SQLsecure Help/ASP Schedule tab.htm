Change snapshot schedule

The Schedule tab allows you to choose the best times to generate a snapshot on your SQL Server instance.

By default, snapshots are scheduled for midnight (of the Repository) every Sunday morning. It is recommended that you schedule snapshots to occur during “off-peak” hours.

How do I use the Schedule tab?

The Schedule tab contains the schedule, if any, that is currently being used for the SQL Server selected instance. To change the schedule, click Change and select the new time and frequency for snapshot collection.

What times should I schedule my snapshot collections?

It is recommended that snapshot collections be scheduled for non-peak or off-hour times. Snapshots are scheduled, by default, for midnight in the local time zone of the SQL Server instance hosting the SQLsecure Repository.

What information is contained on the Schedule tab?

Option

Description

Change

Click this button to edit your audit snapshot collection schedule.

Enable Scheduling

Check this box to enter the audit snapshot schedule.

Keep snapshot for [number] days

Type, or use the up and down arrows, to indicate the number of days that you want to store snapshots in the SQLsecure Repository.
How do I view all snapshot properties?

To view all properties of a snapshot, right-click a snapshot in the Snapshots window and select Properties to view all the properties associated with the snapshot.

How can I view a list of snapshots on my audited SQL Server?

To view a list of snapshots and baselines for a specific SQL Server, select the Audit History tab on the Explore Permissions view.

When is the first snapshot taken?

The first snapshot is taken at the first scheduled snapshot collection time. Snapshots can also be taken manually right-clicking the SQL Server instance in the Explore Permissions view and selecting Collect audit data.
