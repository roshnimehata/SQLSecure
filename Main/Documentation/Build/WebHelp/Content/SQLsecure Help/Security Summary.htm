Assess Your Security Model

The Security Summary view allows you to check the status of your security policies at the enterprise and at the individual monitored SQL Server instance levels. This view includes the overall policy status, a security report card that lists security risks, the settings of each of your SQL Server instances, and the associated user accounts.

What type of information is located on the Enterprise level?

The Security Summary view includes the following at the Enterprise level:

Policy Status

Allows you to view the number of security risks associated with the selected policy and a break down of the risk levels.

Enterprise Security Report Card

Allows you to view all the risks, along with their details. discovered on all the SQL Server instances assigned to the selected policy.

Enterprise Settings

Allows you to view general and security-related settings for your SQL Server instances.

Enterprise Users

Allows you to view the user accounts and account settings for the SQL Server instances assigned to the selected policy.

What type of information is located on the Server level?

The Security Summary view includes the following at the Server level:

Server Status

Allows you to view the number of security risks associated policy and SQL Server instance selected.

Server Security Report Card

Allows you to view all the risks, along with their details, discovered on the SQL Server instance assigned to the selected policy.

Server Settings

Allows you to view general and security-related settings for your SQL Server instances.

Server Users

Allows you to view the user accounts and account settings for the associated SQL Server instance.
