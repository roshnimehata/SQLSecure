Schedule snapshots for routine data collection

The Schedule Snapshots window allows you to choose the best times to collect data from your SQL Server instance. 

By default, snapshots are scheduled to run at midnight every Sunday morning (using the local time of the computer hosting the SQLsecure Repository).

TIP

 SQLsecure requires that a user be logged in as the SQLsecure Administrator to view snapshot schedules.
How do I use the Schedule Snapshots window?

The Schedule Snapshots window contains the data collection schedule that is currently being used for the SQL Server selected instance. To change the schedule, click Change and select the new time and frequency.

 What times should I schedule my snapshot collections?

When possible, schedule snapshots to run during non-peak or off-hour times.

What can I do on the Schedule Snapshots window?
Item Description

Change

Click this button to edit your audit snapshot collection schedule.

Enable Scheduling

Select this option to specify the audit snapshot schedule.

Keep snapshot for [number] days

Specify the number of days that you want to store snapshots in the SQLsecure Repository.
How do I view snapshot properties?

To view the properties of a specific snapshot, right-click the snapshot in the Snapshots window and then select Properties from the context menu.

 How can I view a list of snapshots on my audited SQL Server?

To view a list of snapshots and baselines for a specific SQL Server, select the Audit History tab on the Explore Permissions view.

When is the first snapshot taken?

The first snapshot is taken at the first scheduled snapshot collection time. You can manually take a snapshot at any time by right-clicking the SQL Server instance in the Explore Permissions view and then selecting Take Snapshot from the context menu.
