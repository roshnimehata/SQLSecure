Change connection credentials

The Credentials tab displays the credentials that SQLsecure uses to access the databases on the selected SQL Server instance.

TIP

If the login configuration for the SQL Server you want to audit is case-sensitive, you must enter your login credentials in a case-sensitive format.

 

Option

Description

SQL Server credentials to connect to audited SQL Server

Select SQL Server Agent credentials to use the default credentials of your SQL Server Agent, or click Windows Authentication and enter the credentials in the fields provided.

Windows Credentials to gather Operating System and Active Directory objects

Check the Use same Windows Authentication as above option or specify an alternative account SQLsecure should use to gather information about OS and AD objects.
How do I use the Credentials tab?

The Credentials tab is used for editing the credentials information that SQLsecure uses to access the databases on the selected SQL Server instance. If you need to make changes to your credentials, change the information in the fields provided.
