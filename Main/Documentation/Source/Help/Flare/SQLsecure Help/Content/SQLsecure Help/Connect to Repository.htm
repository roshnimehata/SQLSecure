Connect to the SQLsecure Repository

By default, SQLsecure connects to the Repository when you start the Console. You may need to reconnect to the Repository database under these circumstances:

  • You installed multiple Repository databases
  • You moved the Repository database to another SQL Server instance
  • You lost connection to the SQL Server instance hosting the Repository and must reconnect

Use the Connect to Repository window to specify the SQL Server instance that hosts the Repository database.
